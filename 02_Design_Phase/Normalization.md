# 4. Normalization Report

The database design follows these normalization steps to remove redundancy:

1. **First Normal Form (1NF):** All table attributes are atomic; no repeating groups exist.
2. **Second Normal Form (2NF):** All non-key attributes are fully functional and dependent on the primary key.
3. **Third Normal Form (3NF):** No transitive dependencies exist (non-key columns do not depend on other non-key columns).
